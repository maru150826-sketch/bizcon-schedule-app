# SAMPO QUEST 予定調整アプリ v8

GitHub Pages + Supabaseで動く、ビジコンチーム用の予定調整Webアプリです。

## v8の主な変更

- 投票後に全データを毎回読み直さず、画面へ即反映する軽量版にしました。
- 自動更新を30秒から120秒に変更しました。
- 手動更新ボタンは残しています。
- 候補日ごとに「この日程で確定」と「確定を外す」を切り替えられるようにしました。
- 確定日が複数ある場合も表示できます。
- 確定日のメモ対象を選べるようにしました。
- 場所の選択肢を「Zoom」ではなく「オンライン / 大学」にしました。
- 既に保存されている旧データの「Zoom」は画面上では「オンライン」と表示します。
- 日付欄は固定せず、候補日追加時に毎回選択する形にしました。

## 基本機能

- ログインなしで、回答者一覧から「自分の名前」を選択できます。
- スマホでもPCでも、同じ名前を選べば同じ回答者として投票できます。
- ○ / △ / × を押すだけで回答できます。
- 他の人の回答状況、未回答者、○+△人数を確認できます。
- 選択中の回答者データを削除できます。
- 選択中のメンバーが追加した候補日を削除できます。
- 作業内容は複数選択できます。
- 時間は30分刻みです。
- 場所は オンライン / 大学 の2択です。

## 注意

ログイン機能はありません。そのため、チーム内の誰でも他の人の名前を選べます。
4人程度の身内利用を前提に、操作を簡単にする設計です。

## GitHub Pagesでの公開

1. このフォルダ内のファイルをGitHubリポジトリにアップロードします。
2. `Settings` → `Pages` を開きます。
3. Branchを `main`、folderを `/root` にします。
4. 公開URLを開きます。
5. 更新後は `Ctrl + F5` で強制更新してください。

## Supabase設定

`app.js` の上部にSupabase URLとpublishable keyを設定します。

```js
const SUPABASE_URL = 'https://dgaveiimlslljluimqxn.supabase.co';
const SUPABASE_ANON_KEY = 'sb_publishable_...';
```

`service_role` や `sb_secret_...` は絶対に入れないでください。

## Supabase SQL

Supabaseの `SQL Editor` で `supabase-schema.sql` の中身をすべて貼り付けて `Run` してください。
既存テーブルがあっても再実行できるようにしています。

## 使い方

1. アプリを開く
2. 回答者一覧から自分の名前を選ぶ
3. 名前がない場合だけ、新しい回答者として追加する
4. 候補日を追加する
5. 各候補日に対して ○ / △ / × を押す
6. ○+△が多い候補を確定する
7. 間違って確定した場合は「確定を外す」を押す
8. 確定後、ToDo・決定事項・宿題をメモする
